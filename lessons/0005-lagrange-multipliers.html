0005 · Lagrange 乘数法（有约束优化）
Lagrange Multipliers — Constrained Optimization

基于 Lecture 2/4 §2.3 (Lin, JNU) & Meerschaert §2.3 · 预计阅读 25 分钟

1. Why constraints? / 为什么需要约束优化？

TV problem solution: x₁=4735, x₂=7043, total=11778. But the assembly plant has a capacity constraint of 10,000 total sets. Our unconstrained solution violates this! / 无约束解违反了总产能限制（x₁+x₂ ≤ 10000）。

Constrained optimization problem / 有约束优化问题：
Maximize f(x₁, x₂), subject to g(x₁, x₂) = c (equality constraint)
最大化目标函数，但在约束条件限制的区域内。

2. Geometric intuition / 几何直觉

The constraint g(x₁,x₂)=c defines a curve (the feasible set / 可行域).
We can only move along this curve. Where is f maximized?

Key insight / 关键洞察：At the optimal point, ∇f must be perpendicular to the constraint curve.
Why? If ∇f had a component tangent to the curve, moving along that direction would increase f.

Since ∇g is also perpendicular to the constraint curve (gradient ⟂ level set),
∇f and ∇g must be parallel:
∇f = λ·∇g

This λ is the Lagrange multiplier / Lagrange 乘数.

3. The Lagrange function method / Lagrange 函数方法

Original problem / 原问题：maximize f(x₁, x₂), subject to g(x₁, x₂) = c

Lagrange function / Lagrange 函数：
L(x₁, x₂, λ) = f(x₁, x₂) − λ·[g(x₁, x₂) − c]

Solve / 求解（n+k 个方程，n+k 个未知数）：
∂L/∂x₁ = 0 → ∂f/∂x₁ − λ·∂g/∂x₁ = 0
∂L/∂x₂ = 0 → ∂f/∂x₂ − λ·∂g/∂x₂ = 0
∂L/∂λ = 0  → g(x₁, x₂) − c = 0  (this is just the constraint / 就是原约束!)

This is exactly ∇f = λ∇g together with g = c.
λ's economic meaning / λ 的经济含义 — "shadow price / 影子价格"：
λ = df*/dc — how much the optimal objective value changes when the constraint is relaxed by one unit.
λ tells you "how much more profit would I make if I had one more unit of capacity?"

Multiple constraints: k constraints → k multipliers λ₁,…,λₖ.
Prerequisite: ∇g₁,…,∇gₖ must be linearly independent / 必须线性无关。

4. Handling inequality constraints / 处理不等式约束

Practical approach / 实用方法：
1. Solve unconstrained version first / 先解无约束版本
2. Check if solution satisfies all inequality constraints / 检查是否满足不等式
3. If yes → done! / 满足就结束了
4. If some constraints are violated → set those to equality (they "bind") and use Lagrange / 被违反的约束取等号，用 Lagrange 法
5. Compare all feasible candidates and select the best

Why equality? If a constraint is "slack" (not binding), it doesn't affect the optimum.
The optimum occurs where binding constraints are tight.
